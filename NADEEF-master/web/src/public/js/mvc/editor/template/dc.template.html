Denial Constraints (DCs) define rules between one or more tuples over one or more relations. The syntax requires a conjunction of predicates, where each predicate has two atoms (variable over tuple.attribute or constant) and an operator (=,!=, <,>): The syntax of DC Rule may look like following and a formal definition is described here.

not( t1.Column A > t2.Column B & t1.Column C = constant )

For example, rule not(t1.C=t2.C&t1.A!=t2.A) indicates that there should be no pair of tuples where the C attribute has the same value but has different value on A attribute.
